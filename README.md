# All/App ★

**29の収益化プラットフォーム** のアカウントをまとめて管理する、
マルチプラットフォーム運用デスクトップアプリ。
同じネタを複数プラットフォームに展開して収益化する運用を支援します。

## 対応プラットフォーム (6カテゴリ・29サイト)

| カテゴリ | プラットフォーム |
|---|---|
| SNS・テキスト | X (Twitter) / Threads / Instagram / Pinterest / LINE VOOM |
| 動画 | YouTube / TikTok / ニコニコ動画 / Rumble / Dailymotion |
| ライブ配信 | イチナナ (17LIVE) / Twitch / Pococha / SHOWROOM |
| コンテンツ販売・ブログ | note / Brain / Tips / Kindle出版 (KDP) / BOOTH / ココナラ / FC2ブログ / Medium / Substack |
| メンバーシップ・支援 | pixivFANBOX / Fantia / Ci-en / Patreon |
| 音声 | stand.fm / Spoon |

## アフィリエイト (ASP) 連携 — 「アフィリ」タブ

- 主要ASP 9社を収録: **A8.net / もしもアフィリエイト / バリューコマース /
  afb / アクセストレード / 楽天アフィリエイト / Amazonアソシエイト /
  インフォトップ / Google AdSense** (それぞれの得意分野メモ付き)
- ダブルクリックでASP管理画面がアプリ内ブラウザで開き、ログインは保持されます
- 取得したアフィリエイトリンクは「マイリンク集」に登録 →
  記事作成パネルの「リンク集から選択…」で1クリック挿入

## 起動方法

**PC (デスクトップ版):** `run.bat` をダブルクリック (または `python main.py`)

**外出先から使う (Web公開版):**
1. PCで `run_web.bat` をダブルクリック (または `python web.py`)
   - 初回のみCloudflare公式のトンネルクライアント (約60MB) を自動取得します
2. 画面に表示される `https://～.trycloudflare.com` のURLをiPhoneで開く
   (QRコード画像も自動で開くので、iPhoneのカメラで読み取るのが早い)
3. 8桁の**ログインPIN** (起動画面に表示 / `~/.allapp/mobile_pin.txt`) を入力
4. 共有ボタン → 「ホーム画面に追加」

- データはPCの中のまま。PCが起動している間だけアクセスできます
- URLは起動のたびに変わります (固定したい場合はCloudflareアカウントで
  named tunnel を設定するか、Tailscale をPC/iPhoneに入れる方法もあります)
- HTTPSなのでコピー機能・PWA機能が完全動作します

**iPhone / スマートフォン (自宅Wi-Fi内のみ・モバイル版):**
1. PCで `run_mobile.bat` をダブルクリック (または `python mobile.py`)
   - 初回はWindowsファイアウォールの確認が出るので「アクセスを許可」を押す
2. iPhoneをPCと同じWi-Fiに接続する
3. 画面に表示されたURL (例 `http://192.168.x.x:8787`) をSafariで開く
4. 共有ボタン → **「ホーム画面に追加」** で逆さ星アイコンのアプリになります

モバイル版はデスクトップ版と**同じデータベースを共有**するため、
どちらで操作しても、アカウント・実績・下書き・アフィリリンクは常に同期されます。
デスクトップ版と同時起動も可能です。

| 機能 | PC版 | モバイル版 |
|---|---|---|
| ネタ収集 (バズ上位3件) | ○ | ○ (同じエンジン) |
| 記事作成プロンプト生成 | ○ | ○ |
| AIで作成 | アプリ内ブラウザ | コピーしてAIサイト/アプリへ |
| 実績・アカウント・アフィリ管理 | ○ | ○ |
| SNSへの手動ログイン | アプリ内ブラウザが保持 | Safariが保持 (アカウントタブの「サイトを開く」から) |
| 投稿 | アプリ内ブラウザで手動 | 「投稿画面を開く」で各サイトの投稿画面へ直行。X/Threadsは投稿文が自動セットされる |

必要ライブラリ (インストール済み):

```
pip install -r requirements.txt
```

## 機能

### 1. プラットフォーム切替 (上部プルダウン)
ブラウザのタブのように上部のプルダウンで切り替えます (カテゴリ別に整理)。
初回切替時はブラウザの生成に数秒かかります (2回目以降は即時)。

### 2. アプリ内ブラウザ + 手動ログイン (凍結リスク回避)
- 右側がアプリ内ブラウザ。**APIは一切使いません**
- 各プラットフォームに普通のブラウザと同じように手動ログインしてください
- ログイン情報 (Cookie等) はプラットフォームごとに
  `C:\Users\<ユーザー>\.allapp\profiles\` に保存され、
  **アプリを再起動してもログインは保持されます**
- 投稿も右側のブラウザから通常の手動操作で行います

### 3. ネタ収集 (左パネル) — バズっている「投稿そのもの」を上位5件表示

| プラットフォーム | 取得方法 | 表示内容 |
|---|---|---|
| X | Yahoo!リアルタイム検索 (話題順) | バズ投稿5件 (いいね数・RT数・時刻付き)。キーワード空欄ならトレンド1位の投稿 |
| YouTube | 急上昇ワード×今週×再生数順 | 動画5件 (3日以内・10万再生以上を優先) |
| ニコニコ動画 | 24hランキング | 上位5件 (再生数・コメント付き) |
| Instagram / Threads / Pinterest / TikTok / note / Medium 他多数 | キーワード検索 (Yahoo!検索→DDGの2段構え) | そのキーワードの投稿・人気アカウント上位5件 |
| ライブ配信系 (17LIVE等) | フォールバックボタンで手動確認 | ― |

- X・YouTubeはキーワード空欄でOK (トレンド1位を自動使用)。
  自分のジャンル (例:「ダイエット」「副業」) を入れるとそのジャンルに絞れます
- Instagram等のキーワード検索型は**キーワード必須** (入力欄に案内が出ます)
- 「このネタで作成→」ボタンでそのまま記事作成パネルへ

### 4. 記事作成 (左パネル)
- ネタ・種類 (文章投稿 / 動画台本 / 画像生成プロンプト / アフィリエイト記事)・
  アフィリエイトURLを入れて「プロンプトを生成」
- プラットフォーム別の最適ルール (文字数・ハッシュタグ数・構成) が自動で入ります
- 「Claudeで作成」「ChatGPTで作成」でプロンプトが**自動コピーされた状態で**
  アプリ内ブラウザにAIが開くので、貼り付け (Ctrl+V) て生成
- 生成結果は下書きとして保存可能。「SNSに戻る」ボタンでSNS画面へ復帰

### 5. 実績 (左パネル)
- プラットフォームごとに投稿の閲覧数 / 再生数 / 販売数 / 収益を記録
- 列名はプラットフォームに合わせて自動で変わります
  (note なら「販売数 / 売上」、YouTube なら「再生数 / 高評価」等)
- セルを直接編集して保存。下部に合計を表示

## データの保存場所

| データ | 場所 |
|---|---|
| ログインセッション | `~/.allapp/profiles/<プラットフォーム>/` |
| ASPログイン | `~/.allapp/profiles/asp/` |
| AIツールログイン | `~/.allapp/profiles/ai-tools/` |
| アカウント・実績・下書き・アフィリンク | `~/.allapp/allapp.sqlite3` |

ログインをリセットしたい場合は該当プラットフォームのフォルダを削除してください。

## 注意事項

- **自動投稿は行いません**。ネタ収集と記事作成を自動化し、
  最終投稿は手動にすることでアカウント凍結リスクを最小化する設計です
- トレンド自動取得は各サイトの公開ページを利用しているため、
  サイト側の仕様変更で取得できなくなる場合があります。
  その場合もフォールバックボタンからトレンドページを開けます
- アフィリエイト記事はステマ規制 (景品表示法) 対応のため、
  プロンプトに PR表記 を入れる指示が自動で含まれます

## モバイル版・Web公開版の注意

- **PIN認証**: モバイル版・Web公開版ともに8桁PINでログインが必要です。
  PINは `~/.allapp/mobile_pin.txt` に保存されており、書き換えれば変更できます。
  一度ログインすれば90日間は再入力不要です
- `run_mobile.bat` は同じWi-Fi内のみ、`run_web.bat` はインターネット経由でも
  アクセス可能 (どちらもPC起動中のみ)
- Wi-Fi内アクセス (http) ではSafariの仕様上コピーが失敗することがあります。
  その場合はプロンプト欄を長押しして手動コピーしてください
  (Web公開版はhttpsなので確実に動作します)

新しいプラットフォームを追加するには `config.py` の `PLATFORMS` に
1エントリ追加し、必要なら `prompts.py` の `PLATFORM_RULES` にルールを足すだけです。
